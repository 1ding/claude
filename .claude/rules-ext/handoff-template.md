# 跨会话交接模板

当容量管理判定为"不足"需要中断当前会话时，除将工作文件写入 drafts/ 和 plans/ 外，还须生成交接文件。

## 交接文件位置

`projects/{项目}/plans/交接_{文档名}_{日期}.md`

## 交接文件格式

```markdown
# 交接记录：{文档名}

## 会话信息

- 日期：{YYYY-MM-DD}
- 任务类型：{编写/修订}
- 中断原因：{容量不足/用户主动中断}

## 已完成内容

- {已完成的章节或修订项列表}
- 当前文件位置：`drafts/{文件名}.md`

## 进行中内容

- {正在处理但未完成的章节或修订项}
- 中断点：{具体到哪一节、哪一步}

## 本次决策记录

| 决策点 | 决策内容 | 依据 |
| --- | --- | --- |
| {如：术语选择} | {如：统一使用"限界上下文"} | {如：与项目术语规范一致} |

## 开放问题

- {待用户确认的矛盾或歧义}
- {标注为"逻辑推测"的内容，需用户验证}

## 生效假设

- {本次工作中采用的假设，下次会话需沿用或推翻}

## 下次会话指引

1. 加载文件：`drafts/{文件名}.md` + `plans/{计划文件名}.md` + 本交接文件
2. 从 {具体断点位置} 继续
3. 优先处理：{开放问题中最紧迫的项}
```

## 使用时机

- **生成时机**：容量状态转为"不足"时，与 drafts/ 和 plans/ 文件同步生成
- **恢复时机**：用户在新会话中说"请按 plans/交接_XXX.md 继续"时，读取交接文件恢复上下文
- **与计划文件的关系**：计划文件记录任务分解和进度；交接文件记录决策、假设和语义上下文。恢复时同时加载两者
