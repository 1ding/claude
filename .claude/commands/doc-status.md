# 查看项目状态

查看指定文档项目的当前状态，包括文档清单、编写/修订进度、待处理事项。

## 输入

$ARGUMENTS

## 执行步骤

### 1. 识别项目

从用户输入中识别项目名。若未指定，列出 `projects/` 下所有项目供选择。

### 2. 读取项目配置

读取 `projects/{项目}/CLAUDE.md`，获取：
-   所属领域
-   文档关联关系表

### 3. 扫描项目目录

扫描以下目录，汇总文件状态：

| 目录       | 扫描内容                     |
| ---------- | ---------------------------- |
| `refs/`    | 参考材料列表                 |
| `drafts/`  | 编写/修订中的文档            |
| `plans/`   | 大纲、编写计划、修订计划文件 |
| `outputs/` | 已完成的终稿                 |

### 4. 解析进度

对 `plans/` 中的计划文件，读取进度信息：
-   编写计划：已完成 M/N 个编写项
-   修订计划：已完成 M/N 条修订项

### 5. 输出状态报告

```
---项目状态---

项目：{项目名}
所属领域：{领域}

文档状态：
| 文档         | 状态     | 位置                | 进度          | 备注         |
| ------------ | -------- | ------------------- | ------------- | ------------ |
| 需求规格     | 已完成   | outputs/需求规格.md | -             |              |
| 概要设计     | 编写中   | drafts/概要设计.md  | 3/5 编写项    | 第2批已完成  |
| 详细设计     | 大纲就绪 | plans/大纲_详设.md  | -             | 待开始编写   |
| 接口规范     | 未开始   | -                   | -             |              |

参考材料：{N} 份
计划文件：{N} 份

待处理事项：
- {从计划文件中提取的待解决问题}
- {遗留事项}

建议下一步：
- {根据状态给出建议操作和对应的 /doc-xxx 命令}
```
