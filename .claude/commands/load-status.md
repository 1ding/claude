# 查看已加载文档

扫描当前会话，列出所有已加载的文档及其估算 token 用量。

## 用法

```
/load-status
```

无参数。

## 执行步骤

1.   扫描当前会话历史中所有包含 `[LOADED *]` 标记的记录
2.   按加载角色分组整理：

```
已加载文档清单
─────────────────────────────────────────────
角色         文件路径                                    估算 tokens
[target]     projects/{项目}/drafts/{文档名}.md          ~32,800
[upstream]   projects/{项目}/outputs/{文档名}.md         ~25,600
[spec]       {已加载的规范文件路径}                      ~950
[layout]     {已加载的排版规范路径}                      ~780
...
─────────────────────────────────────────────
已加载合计：~60,130 tokens（约占 200K 基准的 30%）
剩余估算：~139,870 tokens
─────────────────────────────────────────────
```

3.   容量状态：
-   < 60%（120K）→ ✅ 容量充裕
-   60%–80%（120K–160K）→ ⚠️ 谨慎加载
-   > 80%（160K）→ 🔴 强烈建议停止加载

> 注：token 估算基于文件字节数（÷3.5），为近似值，实际误差约 ±20%。
> 已加载合计未含固定开销（系统 prompt ~4K + version-management.md ~2K + claude.md ~0.5K = 约 6.5K 起，每轮对话再加 ~500 tokens）。
> 如需含开销的综合容量估算，请使用 `/cap-status`。
