# 局部快修

对文档执行小范围的局部修正，最小化上下文开销。

## 适用场景

-   修正错别字、错误数据、错误表述
-   替换术语（少量、明确的替换）
-   在指定位置补充一段内容
-   删除一段重复或多余的内容
-   调整一个小节的描述

## 不适用场景（请改用对应 skill）

-   需要对标规范文档的修订 → `/doc-revise` 或 `/doc-fmt`
-   需要与关联文档保持一致 → `/doc-align`
-   大范围修订（超过 3 处不相关的修改） → `/doc-revise`
-   结构重组、章节拆分合并 → `/doc-revise`

## 输入

$ARGUMENTS

## 先决条件检查

**在执行任何步骤前，先完成以下检查：**

扫描当前会话历史中的 `[LOADED *]` 记录，确认：
-   [ ] 已存在 `[LOADED target]` 记录（通过 `/load-target` 加载了目标文档）

如未满足，输出以下提示并**立即中止**，不执行后续任何步骤：

> ⚠️ 请先执行 `/load-target /path/to/目标文档` 后再运行本命令

## 执行步骤

### 1. 确认目标文档

从会话历史最近一条 `[LOADED target]` 记录获取目标文档路径（即当前工作文档）。

### 2. 执行修改

-   按用户指令直接修改目标文档内容
-   修改范围严格限定在用户指定的位置，不扩大修改范围

### 3. 创建新版本

按版本管理规范（`.claude/rules/version-management.md`）创建新版本文件：
-   读取当前最新版本的版本号和修订号
-   如果是 R 版本 → 递增 R（如 R02 → R03）
-   如果是常规版本 → 从 R01 开始（如 v0.11 → v0.11_R01）
-   在修订记录中新增 R.xx 条目，修订内容简述本次修改
-   修订人标注为 AI
-   将新版本写入 `drafts/`

### 4. 简要报告

```
---快修完成---

修改内容：{一句话描述}
涉及位置：{章节号}
新版本：drafts/{新文件名}

注意：本次为局部快修，未执行规范检查和关联文档一致性检查。
如需系统性检查，请使用 /doc-review 或 /doc-align。
```

## 不执行

-   **不加载**规范文档（使用会话中已加载的规范）
-   **不加载**关联文档
-   **不执行**容量评估
-   **不执行**质量检查清单
-   **不输出**方案确认（直接修改）
