# 3 正文章节结构

+   战略规划报告正文应包含8个章节，章节顺序固定，不应调整。
+   3.1–3.8节分别定义各章节的编写规范，每节包含章节综述、章节结构、表达规范三个层次。部分章节在章节结构表前后设有编写规则，作为该章节的约束性要求。

## 3.1 第1章 执行摘要

### 3.1.1 章节综述

+   执行摘要是全报告核心结论的压缩版，面向决策层，使读者无需阅读全文即可把握产品战略全貌并做出Go/No-Go判断。
+   本章不产生新信息——每项数据和结论均应来自正文对应章节，不应出现正文中没有的新结论。

### 3.1.2 章节结构

+   第1章结构与内容要点如下。

    | 编号  | 标题         | 内容要点                                         | 表达规范                       |

### 3.1.3 表达规范

+   执行摘要涉及的表达规范较多，其中与正文章节共享的规范在对应章节定义（见上表"表达规范"列的跨章节引用），以下定义执行摘要的表达规范（含公共规范）。

#### 量化对比表

+   规范：用于"传统方式 vs 产品方式"对比的场景。各章节可根据需要调整列名的具体含义，但核心结构应保持一致。

    | 列名     | 说明                                         |
    | -------- | -------------------------------------------- |
    | 对比维度 | 对比的具体维度（如"需求分析周期""人工成本"） |
    | 传统方式 | 传统方式下的现状数据                         |
    | 产品方式 | 使用本产品后的目标数据                       |
    | 提升幅度 | 量化提升（如"缩短70%""降低50%"）             |
    | 实现前提 | 达成该提升的前提条件（不可省略）             |

#### 分期预算表

+   规范：每行对应一个版本期，展示期间投入和累计投入。

    | 列名     | 说明                              |
    | -------- | --------------------------------- |
    | 期数     | 版本期编号（如"P1""P2"）          |
    | 时间范围 | 起止时间（如"2026-Q1 ~ 2026-Q2"） |
    | 期间投入 | 本期投入金额                      |
    | 累计投入 | 截至本期末的累计投入              |
    | 主要目标 | 本期核心交付目标（一句话）        |

#### 项目核心数据表

+   规范：以单表集中呈现项目全局关键指标，使决策者一目了然。

    | 列名 | 说明                                              |
    | ---- | ------------------------------------------------- |
    | 指标 | 指标名称（如"研发周期""总投入""目标客户数"等）    |
    | 数值 | 具体数值或区间，不应模糊表述                      |
    | 说明 | 数值的口径或前提条件（如"含P1–P3三期""首年目标"） |

#### 客户增长预测表

+   规范：按时间维度展示客户数量的增长预测。

    | 列名       | 说明                             |
    | ---------- | -------------------------------- |
    | 时间节点   | 预测时间点（如"P1末""第12个月"） |
    | 目标客户数 | 预期累计客户数量                 |
    | 新增客户数 | 本期新增客户数量                 |
    | 客户类型   | 按客户分层标注主要客户构成       |
    | 关键假设   | 预测所依赖的核心假设             |

#### 财务汇总表

+   规范：集中呈现关键财务指标的分期预测。

    | 列名       | 说明                   |
    | ---------- | ---------------------- |
    | 时间节点   | 预测时间点             |
    | 收入       | 本期收入               |
    | 成本       | 本期成本               |
    | 净利润     | 本期净利润             |
    | 累计净利润 | 截至本期末累计净利润   |
    | 关键假设   | 本期财务预测的核心假设 |

#### 风险摘要表

+   规范：仅摘录P0/P1级风险，每项风险压缩为一行。详细风险分析见3.7节。

    | 列名       | 说明           |
    | ---------- | -------------- |
    | 风险等级   | P0或P1         |
    | 风险名称   | 简短风险名     |

#### 可行性评分表

+   规范：从多维度对项目可行性进行评分，支撑Go/No-Go判断。

    | 列名     | 说明                                        |
    | -------- | ------------------------------------------- |
    | 评估维度 | 维度名称（如"市场可行性""技术可行性"等）    |

## 3.2 第2章 市场分析

### 3.2.1 章节综述

+   从外部视角论证"市场值得进入"：通过痛点分析证明需求真实存在，通过市场定义和容量估算证明市场规模可支撑商业目标，通过窗口期分析证明时机合理。
+   本章结论向后传递：核心痛点为第4章价值主张提供立论基础，客户分层定义为第3章客户分析和3.5章定价策略提供分类基准。

### 3.2.2 章节结构

+   困境还原应以具体场景为基础，避免泛泛描述——读者应能"看见"目标客户遇到的真实困境。
+   市场容量估算应标注数据来源与估算方法，不应有无出处的数据。

+   第2章结构与内容要点如下。

    | 编号  | 标题         | 内容要点                                         | 表达规范             |
    | ----- | ------------ | ------------------------------------------------ | -------------------- |
    | 2.1   | 核心痛点分析 | 按"困境还原→根因分析"逐个展开                    |                      |
    | 2.1.x | 困境x        | 场景还原（流程文本或伪代码）                     | 3.2.3 困境还原模型   |
    |       |              | 困境表现表                                       | 3.2.3 困境表现表     |
    |       |              | 根本原因                                         |                      |
    | 2.1.n | 共同根源     | 多困境归纳为共同根源                             |                      |
    |       |              | 困境→根源→解决方案关系图                         | 3.2.3 困境归因图规范 |
    | 2.2   | 目标市场定义 |                                                  |                      |
    | 2.2.1 | 市场选择逻辑 | 首选场景选择依据表                               | 3.2.3 场景选择依据表 |
    | 2.2.2 | 客户分层     | 分层表（特征、产品形态、客单价、目标）           | 3.2.3 客户分层表     |
    | 2.2.3 | 市场容量估算 | 分客户类型估算潜在规模，标注估算依据（数据来源） | 3.2.3 市场容量估算表 |
    |       |              | 目标渗透率 + 行业基准对比论证合理性              |                      |
    | 2.3   | 市场窗口期   | 窗口形成条件 + 时间判断 + 先发优势               |                      |
    | 2.4   | 增长驱动因素 | 驱动市场增长的核心因素                           |                      |

    +   第2章结构与内容要点

### 3.2.3 表达规范

#### 困境还原规范

+   规范：每个困境采用以下结构还原，使读者能具象感知目标客户的真实处境：

    | 要素     | 说明                                                             |
    | -------- | ---------------------------------------------------------------- |
    | 场景标题 | 简短命名（如"月末合规报告赶工"）                                 |
