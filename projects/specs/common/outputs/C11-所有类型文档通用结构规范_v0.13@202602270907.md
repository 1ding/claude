# 1 目的和范围

+   文档编号：C11
+   文档名称：文档通用结构规范
+   目的：规定所有由AI协助编写或优化的文档的通用结构要求
+   范围：所有由AI协助编写或优化的文档
+   管理要点：文档分类规则、附录分类规则、文档整体结构、总则构成要素、特定附录编写规范、密级标注规范、修订记录规范
+   文档密级：【商密二级】 -- 外发必追责。

# 2 文档分类

+   文档按目的/用途分为规范类、交付类、辅助类三大类。不同类别的文档采用不同的结构要求。

## 2.1 文档分类表

+   文档分类表如下：

    | 目的/用途  | 结构要求              | 典型示例                                               |
    | ---------- | --------------------- | ------------------------------------------------------ |
    | **规范类** | 完整结构（五分区）    |                                                        |
    | 编写规范   | 五分区                | S01战略规划编写规范、S10软件工程文档编写规范的编写规范 |
    | 通用规范   | 五分区                | C01内容表达规范、C02排版格式规范、C11文档通用结构规范  |
    | 过程规范   | 五分区                | N00过程规范                                            |
    | 术语规范   | 五分区                | C99天帷通用术语规范                                    |
    | **交付类** | 完整结构（五分区）    |                                                        |
    | 需求文档   | 五分区                | 宏观需求规格说明书、详细需求规格说明书                 |
    | 设计文档   | 五分区                | 架构设计文档、详细设计文档                             |
    | 战略规划   | 五分区                | 01战略规划报告                                         |
    | 测试报告   | 五分区                | 系统测试报告、验收测试报告                             |
    | **辅助类** | 简化结构（标题+内容） |                                                        |
    | 清单       | 标题+表格             | 检查清单、待办清单、问题清单                           |
    | 记录       | 标题+时间序列条目     | 会议记录、决策记录(ADR)                                |
    | 索引       | 标题+固定格式         | 术语表、缩略语表、参考文献                             |
    | 模板       | 标题+可填充结构       | 文档模板、代码模板                                     |

## 2.2 结构要求说明

+   **完整结构（五分区）**：规范类和交付类文档采用五分区结构（总则、正文、规范性附录、资料性附录、修订记录），按第3节要求编写。
+   **简化结构（标题+内容）**：辅助类文档采用简化结构，至少包含标题和内容，可选包含修订记录，按第4节要求编写。

# 3 完整结构文档的五分区

+   完整结构文档由以下五个分区组成，顺序固定。

    | 分区       | 必备/可选 | 定位                                   |
    | ---------- | --------- | -------------------------------------- |
    | 总则       | 必备      | 文档基本信息                           |
    | 正文       | 必备      | 文档的核心内容                         |
    | 规范性附录 | 可选      | 执行配套——模板、文档骨架               |
    | 资料性附录 | 可选      | 理解辅助——术语、方法论、教学、检查清单 |
    | 修订记录   | 必备      | 版本演进历史，供人工查阅               |

+   总则和正文在文档中均为一级标题（# 级别）。总则固定为"# 1 {总则标题}"，正文从"# 2"开始连续编号。两者在分区表中是并列的逻辑分区，在文档中是连续的一级标题序列。
+   总则和正文为必备分区，修订记录也为必备（见第7节）。规范性附录和资料性附录为可选分区，无相应内容时可省略。

# 4 简化结构文档要求

+   简化结构文档至少包含标题和内容两部分，可选包含修订记录。
+   标题应清晰说明文档的目的和范围。
+   内容按文档类型采用相应的固定格式：
    -   清单类：表格形式
    -   记录类：时间序列条目
    -   索引类：固定格式（如术语表的三层术语体系）
    -   模板类：可填充的结构
+   修订记录可选，如需包含则按第7节要求编写。

# 5 总则编写要求

+   总则是文档的第一个分区，声明文档的基本信息。总则的标题不强制使用"总则"，可根据内容侧重命名（如"适用范围""规范定位"），以利于AI检索。

## 5.1 通用要素

+   所有文档的总则应包含以下通用要素。各类别文档可根据需要补充专有要素（如规范类文档补充适用对象和管理要点，交付类文档补充编写目的和背景等）。

    | 要素     | 必备/可选 | 说明                    |
    | -------- | --------- | ----------------------- |
    | 文档编号 | 必备      | 按编号规范执行          |
    | 文档名称 | 必备      | 按编号规范执行          |
    | 文档密级 | 必备      | 按第8节密级标注规则执行 |

## 5.2 总则模板

+   总则模板：

    ```text
    # 1 {总则标题}

    +   文档编号：{编号}
    +   文档名称：{名称}
    +   文档密级：{密级标注，格式见第8节}
    ```

# 6 附录体系

## 6.1 附录通用规则

+   附录分为规范性附录和资料性附录两类。附录按目的/用途细分为索引类、配套类、辅助类，详见第6.2节附录分类表。
+   附录编号格式为"附录A""附录B"，按字母依次递增。规范性附录排在资料性附录之前。
+   附录标题格式：附录字母编号 +（性质标注）+ 附录名称。规范性附录标注"（规范性）"，资料性附录标注"（资料性）"。
+   格式示例："# 附录A（规范性）模板"、"# 附录B（资料性）术语表"。
+   资料性附录的推荐排列顺序：术语表 → 文档说明。修订记录固定位于所有附录之后。各文档可根据实际情况调整，但修订记录的末尾位置不可变。

## 6.2 附录分类表

+   附录按目的/用途分为索引类、配套类、辅助类三大类。附录分类表如下：

    | 目的/用途  | 性质建议 | 结构形式                        | 格式要求参见 |
    | ---------- | -------- | ------------------------------- | ------------ |
    | **索引类** |          |                                 |              |
    | 术语表     | 资料性   | 固定格式（三层术语体系）        | 第6.3节      |
    | 缩略语表   | 资料性   | 固定格式（表格）                | 第6.4节      |
    | 参考文献   | 资料性   | 固定格式（列表）                | 第6.5节      |
    | **配套类** |          |                                 |              |
    | 裁剪规则   | 规范性   | 结构化（裁剪条件表 + 裁剪建议） | 第6.6节      |
    | 文档骨架   | 规范性   | 固定格式（代码块/模板）         | 第6.7节      |
    | 检查清单   | 资料性   | 固定格式（表格）                | 第6.8节      |
    | 模板集合   | 规范性   | 结构化（多个模板条目）          | -            |
    | **辅助类** |          |                                 |              |
    | 文档说明   | 资料性   | 结构化（定位图 + 层级关系说明） | 第6.9节      |
    | 内容释义   | 资料性   | 自由组织                        | -            |
    | 补充示例   | 资料性   | 自由组织                        | -            |
    | 应用指南   | 资料性   | 自由组织                        | -            |

    +   **性质建议**：推荐的性质，但具体文档可根据实际情况调整。
    +   **结构形式**：
        -   **固定格式**：本规范提供模板，直接填写
        -   **结构化**：有明确的组成部分和顺序要求
        -   **自由组织**：按内容需要自由组织
    +   **格式要求参见**：标注"-"的附录类型无固定格式要求，按内容需要自由组织。

## 6.3 术语表附录编写规范

+   文档宜包含术语表附录。性质为资料性，标题格式："# 附录X（资料性）术语表"。
+   一致性约束：附录编号格式和标题格式以第6.1节为权威来源。
+   术语表按《C01-内容表达规范》定义的三层术语体系组织，模板如下：

    ```text
    # 附录X（资料性）术语表

    +   业界通用术语（AI已具备充分理解能力，仅列出名称）：
        -   {术语1}、{术语2}、…

    +   天帷通用术语（定义参见《C99-天帷通用术语规范》）：
        -   {术语1}、{术语2}、…

    +   文档专有术语：
        -   {术语名}：{定义}
        -   （无专有术语时写"本文档无文档专有术语"）
    ```

+   业界通用术语仅列出名称，不定义。天帷通用术语仅列出名称和来源，不重复定义。文档专有术语在此完整定义。

## 6.4 缩略语表附录编写规范

+   当文档中使用较多缩略语时，宜包含缩略语表附录。性质为资料性，标题格式："# 附录X（资料性）缩略语表"。
+   一致性约束：附录编号格式和标题格式以第6.1节为权威来源。
+   缩略语表按字母顺序排列，模板如下：

    ```text
    # 附录X（资料性）缩略语表

    | 缩略语 | 英文全称                                     | 中文说明          |
    | ------ | -------------------------------------------- | ----------------- |
    | API    | Application Programming Interface            | 应用程序编程接口  |
    | CI/CD  | Continuous Integration/Continuous Deployment | 持续集成/持续部署 |
    ```

## 6.5 参考文献附录编写规范

+   当文档引用外部文献时，应包含参考文献附录。性质为资料性，标题格式："# 附录X（资料性）参考文献"。
+   一致性约束：附录编号格式和标题格式以第6.1节为权威来源。
+   参考文献按引用顺序或字母顺序排列，模板如下：

    ```text
    # 附录X（资料性）参考文献

    +   [1] {作者}. {文献名称}[{文献类型}]. {出版信息}, {年份}.
    +   [2] {标题}. {URL}, {访问日期}.
    ```

## 6.6 裁剪规则附录编写规范

+   当文档内容设有强制性级别标注（如"(必选)""(宜)""(可选)"）时，应在规范性附录中提供裁剪规则。性质为规范性，标题格式："# 附录X（规范性）裁剪规则"。
+   一致性约束：附录编号格式和标题格式以第6.1节为权威来源。
+   裁剪规则附录由裁剪条件和裁剪建议两部分组成，裁剪建议为可选。模板如下：

    ```text
    # 附录X（规范性）裁剪规则

    ## X.1 裁剪条件

    +   {文档名称}各章节内容要点标注了三级强制性级别，各级别的裁剪条件如下表所示。

        | 标注   | 裁剪条件                                           |
        | ------ | -------------------------------------------------- |
        | (必选) | 不可裁剪；如确实不适用 → 保留标题 + 注明不适用原因 |
        | (宜)   | 可直接省略                                         |
        | (可选) | 可直接省略                                         |
        | 无标注 | 同(必选)，确有充分理由不适用时保留标题并注明原因   |

    +   裁剪后仍应确保{关键追溯链或结构完整性要求}。

    ## X.2 裁剪建议（可选）

    +   不可裁剪的章节及理由：
        -   {章节名}：{不可裁剪的理由}
    +   裁剪建议：

        | 项目规模 | 判断参考   | 裁剪策略   |
        | -------- | ---------- | ---------- |
        | 标准项目 | {判断参考} | {裁剪策略} |
        | 小型项目 | {判断参考} | {裁剪策略} |
    ```

## 6.7 文档骨架附录编写规范

+   当规范文档需要为目标文档提供结构模板时，应包含文档骨架附录。性质为规范性，标题格式："# 附录X（规范性）文档骨架"。
+   一致性约束：附录编号格式和标题格式以第6.1节为权威来源。
+   文档骨架用代码块展示完整的章节结构和必要说明，模板如下：

    ```text
    # 附录X（规范性）文档骨架

    +   {目标文档名称}的文档骨架如下：

        ```markdown
        # 1 {章节名}

        {章节说明或必备要素}

        # 2 {章节名}

        ## 2.1 {小节名}

        {小节说明}
        ```
    ```

## 6.8 检查清单附录编写规范

+   当文档需要提供检查或验证清单时，宜包含检查清单附录。性质为资料性，标题格式："# 附录X（资料性）检查清单"。
+   一致性约束：附录编号格式和标题格式以第6.1节为权威来源。
+   检查清单按检查维度或流程顺序组织，模板如下：

    ```text
    # 附录X（资料性）检查清单

    +   {清单用途说明}

        | 检查项    | 检查内容       | 参考章节 |
        | --------- | -------------- | -------- |
        | {检查项1} | {具体检查内容} | 第X.X节  |
        | {检查项2} | {具体检查内容} | 第X.X节  |
    ```

## 6.9 文档说明附录编写规范

+   文档说明附录为可选附录，性质为资料性，标题格式："# 附录X（资料性）文档说明"。当文档在规范体系中有明确的上下位关系或协同关系时，宜包含此附录。
+   一致性约束：附录编号格式和标题格式以第6.1节为权威来源。
+   文档说明附录用于说明本文档在规范体系中的位置——它被哪些上位规范约束、它约束哪些下位文档、与哪些同级文档协同。
+   文档说明附录可采用以下形式之一：
    -   **图文结合**：定位图（ASCII图）+ 层级关系文字说明。适用于规范体系、系列流程类文档等关系复杂的场景
    -   **纯文字**：直接用文字说明上下位关系和协同关系。适用于关系简单的场景
+   无论采用何种形式，均应满足《C01-内容表达规范》的文字自足原则——即使有图，也应有等效的文字说明。
+   本规范附录B为图文结合形式的完整示范。

# 7 修订记录规范

+   所有文档应在末尾包含修订记录，记录版本演进历史。
+   修订记录使用表格格式（标记、日期、修订内容、修订人），供人工查阅。修订内容列单行显示宽度不超过180（中文算2，英文/数字/符号算1），超出时拆为多行，同一版本的多行共享标记和日期。
+   修订记录的标题格式为"# 修订记录"。
+   修订记录按时间正序排列（最早的版本在最上方，最新的版本在最下方）。
+   标记列有两种值：版本号（如0.10、0.25）和临时修订标记R.{nn}（如R.01、R.02）。两种条目共存在同一张表中。
+   临时修订追踪规则：
    +   每次修订在修订记录表中新增一行，标记为R.{nn}（{nn}从01开始顺序递增）。
    +   当文档作者决定不调整版本号时，将所有R.{nn}行归并为一条修订内容，写入表中最后一个版本行，然后删除所有R.{nn}行，{nn}归零。归并时只描述最终结果与该版本原始状态的差异，不体现R.01到R.0n的中间演变过程。
    +   当文档作者决定调整版本号时，新增一个版本行，将所有R.{nn}行归并为一条修订内容写入该新版本行，然后删除所有R.{nn}行，{nn}归零。归并时只描述新版本与上一个正式版本的差异，不体现R.01到R.0n的中间演变过程。
+   版本号格式为"主版本.次版本"（如0.10、1.00）。
+   修订记录模板如下：

    ```text
    # 修订记录

    | 标记 | 日期       | 修订内容   | 修订人 |
    | ---- | ---------- | ---------- | ------ |
    | 0.10 | YYYY-MM-DD | {修订内容} | {姓名} |
    ```

# 8 文档密级标注规则

+   所有文档应在总则中标注文档密级。
+   密级分为三级，按敏感程度从高到低依次为：【商密一级】、【商密二级】、【内部资料】。
+   【商密一级】：核心商业秘密或涉及重大安全的内容，仅限指定人员访问。标注格式："文档密级：【商密一级】 -- 仅限指定人员，外发必追责。"
+   【商密二级】：内部规范、技术方案等不宜外传的内容，限内部人员访问。标注格式："文档密级：【商密二级】 -- 外发必追责。"
+   【内部资料】：一般性内部文档，无特殊限制但不主动对外公开。标注格式："文档密级：【内部资料】 -- 不主动对外公开。"
+   未标注密级的文档默认视为【内部资料】级别。
+   "外发"定义：将文档内容传递给天帷体系外部的任何个人或组织，包括但不限于客户、合作伙伴、外部顾问。天帷体系内部跨团队传递不属于外发。

+   文档专有术语：
    -   外发：将文档内容传递给天帷体系外部的任何个人或组织，包括但不限于客户、合作伙伴、外部顾问。天帷体系内部跨团队传递不属于外发。
    -   完整结构文档：采用五分区结构（总则、正文、规范性附录、资料性附录、修订记录）的文档，适用于规范类和交付类文档。
    -   简化结构文档：采用简化结构（标题+内容，可选修订记录）的文档，适用于辅助类文档。

+   层级关系说明：
    -   **并列协同关系**：C11、C01、C02 三份规范并列，分别从结构、表达、格式三个维度约束所有文档。C11管辖文档的分区结构、附录体系、修订记录等结构性要素；C01管辖正文的语言写法、逻辑性、用语风格、术语处理、信息表达形式选择；C02管辖正文的排版格式
    -   **上下位关系**：M01是C11的下位专用规范——在C11定义的通用分区结构基础上，为规范类文档增加专用的收录标准和编写要求。S10是M01的下位专用规范——为技术文档编写规范增加专用体例约束
    -   **叠加约束**：规范类文档同时受C11（通用结构）、M01（专用要求）的多重约束，其中技术文档编写规范还额外受S10（专用体例）约束。各类型文档均可在C11基础上补充专有要求
