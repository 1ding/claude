# 1 总则

+   文档编号：C02
+   文档名称：排版格式规范
+   适用对象：所有由AI协助编写或优化的 Markdown 文档
+   管辖范围：Markdown 符号使用、目录层级、内容缩进、图表排版、表格格式、代码块与图块格式、公式与特殊格式
+   不管辖的范围：
    -   文档分区结构、总则编写要求、附录体系、修订记录格式、密级标注（见《C11-文档通用结构规范》）
    -   语言写法和用语风格（见《C01-内容表达规范》）
    -   规范类文档的内容组织结构（见《M01-通用规范编写规范》）
+   文档密级：【商密二级】 -- 外发必追责。

# 2 Markdown 符号定义

+   `+` 通用内容行标记，用于段落、说明、描述等所有非列表内容。
+   `-` 列表项标记，仅用于真正的枚举、并列项。禁止用 `*` 作为列表符号。
+   注意：`+` 在本规范中是通用内容行标记，不作为列表符号使用。
+   `#` 标题标记。
+   ``` 代码块或图块的起止标记。
+   4空格为每层缩进单位。
+   除标题（`#`）、列表项（`-`）、代码块/图块（```）外，所有内容行必须用 `+` 开头。
+   禁止使用的 Markdown 语法：引用块（`>`）、脚注（`[^]`）、任务列表（`- [ ]`）、分隔线（`---` 或 `***`，表格分隔行除外）。如有引用需求，应在正文中以文字形式表述来源。

# 3 目录层级结构

+   严格按照六级标题结构组织内容。文档正文从一级标题（`#`）开始，不应在一级标题之上再设文档名称标题。
+   第一级：`# X 目录标题`
+   第二级：`## X.X 目录标题`
+   第三级：`### X.X.X 目录标题`
+   第四级：`#### X.X.X.X 目录标题`
+   第五级：`##### X.X.X.X.X 目录标题`
+   第六级：`###### X.X.X.X.X.X 目录标题`

# 4 内容缩进规则

+   所有内容行必须明确归属，通过缩进体现父子关系。每深一层缩进4个空格。
+   标题下的 `+` 不缩进。
+   其他 `+` 按层级缩进，包括 `+` 和 `-` 下的 `+` 也要缩进（体现父子包含关系）。
+   `-` 列表项的缩进规则：`-` 与其所在层级的 `+` 保持相同缩进深度。`-` 下的子 `-` 或子 `+` 再缩进4空格。
+   `+` 下面可以有 `-`，`-` 下面也可以有 `+`，可自由嵌套。

# 5 图表标题与说明

+   图和表的上方应有简要说明，阐明用途或内容概要，应为完整陈述句，不应以冒号结尾。
+   图和表的标题置于下方。
+   图、表、代码块相对于其说明文字缩进4空格。
+   图、表、代码块一律不编号。引用时应以"上图""下表""以下代码"等位置性语言指代，或直接引用其标题文字。

# 6 表格对齐规则

+   表格必须使用标准 Markdown 表格语法：`|` 分隔列，`---` 定义表头分隔行。禁止使用空格对齐、制表符或其他方式模拟表格。
+   表头行不得留空，每列必须有明确的列标题。
+   通过填充空格确保在等宽字体环境下表格各列严格对齐。
+   表格列宽应适配实际内容，不应过度填充空格。
+   表格内禁止使用 `<br>` 换行。需换行时应拆分为多行，每行作为独立单元格行。
+   表格单行显示宽度不超过360（中文字符算2个宽度单位，英文/数字/符号算1个），超出时拆行处理。

# 7 公式与特殊格式

+   公式使用 LaTeX 语法。行内公式用 `$...$`，独立公式用 `$$...$$`。
+   允许在以下场景使用 HTML 片段：上下标（sup/sub）、文字高亮（mark）、下划线（u）。
+   除上述允许的场景外，禁止在正文中使用其他 HTML 标签。

# 8 代码块格式

+   代码块必须注明语言标识。常见语言标识：json、yaml、python、java、javascript、sql、bash 等。
+   纯文本使用 ```text。
+   当文档内容需要展示代码块语法本身（即嵌套场景）时，外层使用4空格缩进的方式呈现，不得使用反引号嵌套。

# 9 图块格式与ASCII文本图规范

+   图只允许使用文本图/ASCII图，以 ```graph 标记。`graph` 为本规范体系自定义的语言标识，专用于ASCII文本图。
+   ASCII 文本图必须使用制图专用符号（如 `─ │ ┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼ ▶ ▼ ◆ ●` 等），禁止使用普通字符（如 `+ - | *`）模拟图形元素。
+   ASCII 文本图的规范分为尺寸、对齐、元素、布局四组要求。
+   尺寸要求：
    +   文本图单行显示宽度（包含左侧缩进空格）严禁超过360（中文算2，英文/数字/符号算1）。
    +   超宽处理方式一：逻辑换行，将水平流向的流程图改为垂直流向，或利用折行连接线。
    +   超宽处理方式二：分图表达，按逻辑模块拆分为子图。
    +   超宽处理方式三：简化内容，精简框内文字，详细说明移至图下方文字描述。
+   对齐要求：
    +   文本图内部元素应严格对齐。同级元素水平对齐、垂直对齐。连接线应与元素边界对齐。文字应在框内居中或左对齐，保持一致。
    +   同类、同级元素应大小相同。元素宽度应根据内容合理设定，同类元素取最大内容宽度统一。元素高度应保持一致。
+   布局要求：
    +   元素布局应充分利用可用宽度（但显示宽度不超过360）。同级元素应均匀分布、间距相等。
    +   避免元素过度集中或过度分散。整体布局应视觉均衡、重心稳定、留白合理。
